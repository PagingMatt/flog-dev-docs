Module type Consumers.Consumer

Signature for the combinable message consumers which satisfy the monoid laws:

  • Closure: Combining two consumers results in a consumer.
  • Associativity: Must be guaranteed by implementation of the signature.
  • Identity: From the empty/new consumer.

This means that combining two consumers is lightweight and implementing a monadic bind operator at the Logger.t level to combine multiple streams of logging becomes trivial.

Contractually called when the logger is closed, this optionally can return the state held by the logger or can just be used to side-effect and guarantee the flushing of messages held in the consumer.

returns
