Module Consumers.ConsoleConsumer

Simple message consumer which synchronously drops messages out onto the stdout stream. Given that consumption is synchronous there is no underlying state held in the consumer and combination is associative by design.

type t

Consumer type.

val empty : t

Consumer identity value.

returns

An 'empty' or 'new' consumer which behaves as the identity value for the Consumer monoid.

val combine : t -> t -> t

Associative combiner function.

returns

The associative combination of the two Consumers.

val consume : t -> Message.t -> t

Function to drive consumption of messages.

returns

The consumer which is the result of the inital consumer consuming the message passed in.

val flush : t -> Message.t list option

Contractually called when the logger is closed, this optionally can return the state held by the logger or can just be used to side-effect and guarantee the flushing of messages held in the consumer.

returns

The state inside the consumer if it exists.
